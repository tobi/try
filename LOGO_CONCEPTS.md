# Logo Concepts for `try`

This document captures logo design concepts proposed by the community for the `try` project.

> **Source:** GitHub Issue [#26](https://github.com/tobi/try/issues/26)  
> **Contributor:** Community design contribution  
> **Status:** Under consideration

---

## Design Philosophy

The `try` project is built for developers with ADHD, embracing "beautifully chaotic minds" and "organized chaos." Any logo should reflect:

- **Developer-centric aesthetic** - Terminal/CLI focused
- **Minimalism** - Clean, confident, uncluttered
- **Experimentation** - The spirit of trying things out
- **Organization from chaos** - Finding clarity in scattered ideas

---

## Concept 1: "Minimal File Path"

### Description
A typography-based logo inspired by file paths and terminal prompts.

### Visual Design
```
t/r/y >
```

The design uses:
- Lowercase, clean font
- Forward slashes (`/`) evoking file system paths
- Terminal prompt character (`>`) integrated into the design
- Monospace or developer-friendly typeface

### Rationale
- **Developer-centric:** Speaks directly to the target audience
- **Confident & minimal:** No unnecessary decoration
- **Functional aesthetic:** Looks like something you'd type in a terminal
- **Memorable:** Simple enough to reproduce in ASCII art or text

### Use Cases
- Terminal splash screens
- ASCII art in help text
- Minimal header for documentation
- Social media profile images

## Concept 2: "Scribble"

### Description
A visual metaphor showing chaos contained and transformed into clarity.

### Visual Design
A clean geometric shape (circle or square) containing:
- **Inside:** Tangled, chaotic scribble or winding path
- **Emerging:** Single straight line with arrowhead extending from the shape

### Rationale
- **Powerful metaphor:** Messy experiments â†’ organized clarity
- **Visual storytelling:** Shows the transformation `try` provides
- **Memorable:** Unique visual that stands out
- **Flexible:** Works at various sizes and contexts

### Symbolism
- **Chaotic scribble:** Scattered experiments and ideas
- **Container:** `try` giving them a place and structure
- **Emerging line:** The clarity and focus the tool provides
- **Arrow:** Direction, progress, the idea becoming your next project

### Variations
The scribble can be:
- Contained within a geometric shape with an emerging line
- Integrated within the letterforms of the word "try"
- Animated (for digital contexts) showing the transformation

---

## Concept 3: "Scribble in Typography"

### Description
A variation of Concept 2 where the chaotic scribble is contained within the word "try" itself.

### Visual Design
The letters `t`, `r`, `y` with:
- Clean, bold letterforms
- Chaotic scribble/tangled lines contained within the letter shapes
- Optional: A clean line emerging from the final letter

### Rationale
- **Combines typography with metaphor:** Best of both concepts
- **Readable yet meaningful:** Still clearly says "try"
- **Unique:** Distinctive visual identity
- **Scalable:** Works as both icon and wordmark

## Implementation Guidelines

### File Formats Needed
- **SVG:** Primary format for scalability
- **PNG:** Multiple sizes (16x16, 32x32, 64x64, 128x128, 256x256, 512x512)
- **ICO:** For favicon/Windows icons
- **ASCII/Unicode:** Text-based version for terminal display

### Color Palette Suggestions
Given the terminal/CLI nature:
- **Primary:** Monochrome (black/white) for versatility
- **Accent:** Terminal green (`#00FF00`) or amber (`#FFBF00`)
- **Dark mode:** Ensure visibility on dark backgrounds
- **Light mode:** Ensure visibility on light backgrounds

### Typography Recommendations
For Concept 1 and 3:
- Monospace fonts: `JetBrains Mono`, `Fira Code`, `Source Code Pro`
- Clean sans-serif: `Inter`, `SF Mono`, `IBM Plex Mono`
- Terminal-style: `Courier New`, `Monaco`, `Consolas`
